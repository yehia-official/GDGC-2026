MODULE 11

🎬 Multimedia Elements

HTML5 brought native support for video, audio, and embedded content — no plugins required. These elements make the web rich and engaging.

🎥

<video>

Embeds a video player. Supports controls, autoplay, loop, and muted attributes.

🎵

<audio>

Embeds an audio player. Same attributes as <video>. Supports MP3, OGG, WAV formats.

🔊

<source>

Provides multiple file formats inside <video> or <audio>. Browser picks the first supported one.

🖼️

<iframe>

Embeds another HTML page or external content (YouTube, Google Maps, etc.) inside your page.

🎨

<canvas>

A drawable surface controlled by JavaScript. Used for animations, games, and data visualizations.

🗺️

<picture>

Responsive images — serve different image sizes or formats depending on screen size or browser support.

Video, Audio & iFrame
<!-- Video with controls -->
<video controls width="640" poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support video.
</video>

<!-- Audio player -->
<audio controls>
  <source src="song.mp3" type="audio/mpeg">
</audio>

<!-- Embed YouTube video -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560"
  height="315"
  allowfullscreen
></iframe>

<!-- Responsive image with fallback -->
<picture>
  <source srcset="img.webp" type="image/webp">
  <img src="img.jpg" alt="Description">
</picture>

🔣 HTML Entities

Some characters have special meaning in HTML — like < and >. Entities let you display them safely, plus add special symbols that don't exist on a keyboard.

💡
Why Entities? If you type <p> inside your content, the browser thinks it's a tag. Write &lt;p&gt; to show it as visible text.
📝

Named Entities

Human-readable like &amp;, &copy;, &nbsp;

🔢

Numeric Entities

Decimal &#169; or hex &#x00A9; — both produce ©

🚫

Non-breaking Space

&nbsp; prevents line-breaking between words.

🔍 Meta Tags & SEO

Meta tags live in <head> and are invisible to users — but they power search engine results, social media previews, and browser behavior.

Essential Meta Tags
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Learn HTML from scratch.">
  <meta name="author" content="Yehia">
  <!-- Open Graph (Facebook/LinkedIn) -->
  <meta property="og:title" content="HTML Masterclass">
  <meta property="og:description" content="Interactive HTML course">
  <meta property="og:image" content="https://example.com/preview.jpg">
  <!-- Twitter Card -->
  <meta name="twitter:card" content="summary_large_image">
  <!-- Canonical URL -->
  <link rel="canonical" href="https://example.com/page">
  <!-- Favicon -->
  <link rel="icon" href="favicon.ico">
</head>

♿ Accessibility (a11y)

Accessibility means making your website usable by everyone — including people who use screen readers, keyboard navigation, or have visual impairments.

🌍
1 Billion People Over 1 billion people worldwide live with some form of disability. Accessible websites are good practice and legally required in many countries.
🏷️

aria-label

Provides an accessible name for elements that lack visible text — like icon buttons.

👁️

aria-hidden

Hides decorative elements from screen readers so they don't clutter the experience.

🎭

role

Assigns a semantic role — role="button", role="alert", role="dialog".

⌨️

tabindex

Controls keyboard tab order. 0 = natural order, -1 = JS only.

🔤

Meaningful alt

Always provide meaningful alt text. Empty alt="" tells screen readers to skip decorative images.

🔗

lang

<html lang="en"> tells screen readers which language to use for pronunciation.

Accessibility Examples
<!-- ✅ Icon button with label -->
<button aria-label="Close menu">
  <span aria-hidden="true">✕</span>
</button>

<!-- ✅ Decorative image -->
<img src="divider.svg" alt="" aria-hidden="true">

<!-- ✅ Input with description -->
<label for="email">Email</label>
<input type="email" id="email"
       aria-describedby="hint">
<span id="hint">We never share your email.</span>

<!-- ❌ Bad: No alt, div as button -->
<img src="btn.png">
<div onclick="submit()">Click me</div>

✅ Form Validation

HTML5 has built-in validation attributes that check user input before it's submitted — no JavaScript needed for the basics.

🚨

required

Field must be filled before form submits. Browser shows an error automatically.

📏

minlength / maxlength

Minimum and maximum number of characters allowed.

🔢

min / max

For number/date inputs — sets the allowed value range.

🔍

pattern

A RegExp the input must match — e.g. phone number format.

📧

Type Validation

type="email", type="url" automatically validate format.

📦

fieldset + legend

Groups related form controls with a visible caption — great for radio button groups.

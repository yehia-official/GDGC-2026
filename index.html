MODULE 01

What is HTML?

HTML is the skeleton of every webpage. Before any design, before any logic — there is HTML. Let's understand what it actually is.

🧱

HyperText

Text that links to other text — the original superpower of the web.

📐

Markup

Special codes that annotate content, telling browsers what things are.

🌐

Language

A standardized set of rules browsers understand worldwide.

💡
Key Insight HTML is not a programming language — it has no logic or conditions. It is a markup language: it describes the structure and meaning of content.

HTML was invented by Tim Berners-Lee in 1991 and has evolved into HTML5, the standard in use today. Every website, from Google to Netflix, is built on HTML as its foundation.

MODULE 02

HTML Document Structure

Every valid HTML file follows a strict boilerplate. Understanding this skeleton is step one for every web developer.

index.html
<!-- 1. The DOCTYPE Declaration -->
<!DOCTYPE html>

<!-- 2. The root HTML element -->
<html lang="en">

  <!-- 3. The HEAD (metadata, invisible) -->
  <head>
    <meta charset="UTF-8">
    <title>My Webpage</title>
  </head>

  <!-- 4. The BODY (visible content) -->
  <body>
    <h1>Hello, World!</h1>
  </body>

</html>
📣

<!DOCTYPE html>

Tells the browser to use the modern HTML5 standard. Always the very first line.

🌍

<html>

The root element. Every other element lives inside it. Use lang for accessibility.

🧠

<head>

Metadata — title, character set, linked files. Nothing in <head> is visible to users.

👁️

<body>

Everything the user sees. All your content — text, images, links — lives here.

🌳
The DOM Tree HTML creates a Document Object Model (DOM) — a tree of nested elements. Browsers read this tree to render your page.
document
└─ <html>
├─ <head>
│ ├─ <meta> charset="UTF-8"
│ └─ <title> "My Webpage"
└─ <body>
└─ <h1> "Hello, World!"

⚡
Self-Closing Tags Some elements have no content and don't need a closing tag. These are called void elements: <br>, <img>, <input>, <meta>, <link>

Elements can also be nested — placed inside other elements. The outer element is the parent, the inner ones are children. This nesting creates the DOM tree.

Nested Elements
<!-- Nesting: p is PARENT, strong is CHILD -->
<p>
  This is <strong>very important</strong> text.
</p>

<!-- ✅ Correct — properly closed -->
<em><strong>Bold & Italic</strong></em>

<!-- ❌ Wrong — tags overlap (invalid HTML) -->
<em><strong>Bad!</em></strong>

<h1>

The top-level heading. There should be only one <h1> per page. Critical for SEO and accessibility screen readers.

<h1>Welcome to My Website</h1>
🎯
Semantic vs Visual Use tags for their meaning, not appearance. <strong> means "important content" — it happens to look bold. <em> means "emphasized" — it appears italic. Never choose tags just for how they look!

MODULE 05

HTML Lists

Lists are everywhere on the web — navigation menus, bullet points, steps in a recipe. HTML provides three types.

🔵

<ul> Unordered

A bulleted list. Use when order doesn't matter — a list of features, ingredients, etc.

🔢

<ol> Ordered

A numbered list. Use when sequence matters — steps, rankings, instructions.

📖

<dl> Description

A list of term/definition pairs. Ideal for glossaries, metadata, or FAQs.

All Three List Types
<!-- Unordered list -->
<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>

<!-- Ordered list -->
<ol>
  <li>Open editor</li>
  <li>Write HTML</li>
  <li>Save and preview</li>
</ol>

<!-- Description list -->
<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
</dl>

MODULE 06

Links & Images

Links connect the web. Images bring it to life. These two elements make up the majority of web content.

Links & Images
<!-- Anchor tag: creates a hyperlink -->
<a href="https://example.com">
  Click to visit Example
</a>

<!-- Open in a new tab -->
<a href="https://example.com" target="_blank">
  Open in New Tab
</a>

<!-- Image: a void element -->
<img
  src="photo.jpg"
  alt="A beautiful sunset"
  width="400"
>

<!-- Linked image -->
<a href="https://example.com">
  <img src="logo.png" alt="Logo">
</a>
🔗

href Attribute

Stands for Hypertext REFerence. Holds the URL or path the link navigates to.

🖼️

src Attribute

The image source — a URL or file path. Without it, nothing displays.

♿

alt Text

Alternative description for screen readers and when the image fails to load. Always include it!

🪟

target="_blank"

Opens the link in a new browser tab. Useful for external links so users don't leave your site.

HTML Attributes

Attributes are extra information added to elements. They modify behavior, link resources, and give elements identity.

id

A unique identifier for one element. Like a person's ID number — no duplicates.

🏷️

class

A reusable label you can apply to multiple elements. Primarily used to target CSS styles.

🎨

Inline CSS. Quick but not ideal for large projects. Good for learning and prototyping.

Adds a tooltip shown on hover. Useful for extra information or icon labels.

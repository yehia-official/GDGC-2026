🌐 What is HTML?

HTML is the skeleton of every webpage. Before any design, before any logic — there is HTML.

🔤

HyperText

Text that links to other text — the original superpower of the web.

📐

Markup

Special codes that annotate content, telling browsers what things are.

💬

Language

A standardized set of rules browsers understand worldwide.

💡
Key Insight HTML is not a programming language — it has no logic or conditions. It is a markup language that describes the structure and meaning of content.

HTML was invented by Tim Berners-Lee in 1991. Today's standard is HTML5, which every modern browser supports. Every website — from Google to Netflix — is built on HTML as its foundation.

🏗️ HTML Document Structure

Every valid HTML file follows a strict boilerplate. Understanding this skeleton is step one for every web developer.

index.html
<!-- 1. DOCTYPE Declaration -->
<!DOCTYPE html>

<!-- 2. Root HTML element -->
<html lang="en">

  <!-- 3. HEAD: metadata, invisible -->
  <head>
    <meta charset="UTF-8">
    <title>My Webpage</title>
  </head>

  <!-- 4. BODY: visible content -->
  <body>
    <h1>Hello, World!</h1>
  </body>

</html>
📣

<!DOCTYPE html>

Tells the browser to use HTML5. Always the very first line.

🌍

<html>

The root element. Every other element lives inside it.

🧠

<head>

Metadata — title, charset, linked files. Nothing here is visible to users.

👁️

<body>

Everything the user sees — text, images, links.

🌳
The DOM Tree HTML creates a Document Object Model (DOM) — a tree of nested elements. Browsers read this tree to render your page.
document
└─ <html>
├─ <head>
│ ├─ <meta> charset="UTF-8"
│ └─ <title> "My Webpage"
└─ <body>
└─ <h1> "Hello, World!"

📋 HTML Lists

Lists are everywhere on the web — navigation menus, steps, bullet points. HTML provides three types.

🔵

<ul> Unordered

A bulleted list. Use when order doesn't matter — features, ingredients, etc.

🔢

<ol> Ordered

A numbered list. Use when sequence matters — steps, rankings, instructions.

📖

Term/definition pairs. Ideal for glossaries, metadata, or FAQs.

📝 HTML Forms

Forms are how users interact with websites — login, search, signup, checkout. Every form you've ever filled in was built with HTML form elements.

📦

<form>

The container that wraps all form controls. Has action (where to send data) and method (GET or POST).

⌨️

<input>

The most versatile form element. Its behavior changes based on the type attribute.

🏷️

<label>

Describes an input to the user. Linking it with for improves accessibility.

📋

<textarea>

Multi-line text input. Perfect for messages, comments, or long-form content.

🔽

<select>

A dropdown list. Contains <option> elements for each choice.

🚀

<button>

A clickable button. Use type="submit" to send form data, or type="button" for JS actions.

Complete Form Example
<form action="/submit" method="POST">

  <!-- Text input -->
  <label for="name">Full Name</label>
  <input type="text" id="name" placeholder="Your name">

  <!-- Email input -->
  <label for="email">Email</label>
  <input type="email" id="email" required>

  <!-- Dropdown -->
  <select name="country">
    <option value="eg">Egypt</option>
    <option value="us">USA</option>
  </select>

  <!-- Textarea -->
  <textarea rows="4" placeholder="Your message..."></textarea>

  <!-- Checkbox -->
  <input type="checkbox" id="agree">
  <label for="agree">I agree to the terms</label>

  <!-- Submit button -->
  <button type="submit">Send</button>

</form>

Input Types

The type attribute completely changes how an <input> behaves. Click any type to see it:

♿
Always Link Labels to Inputs Use for="inputId" on the label and matching id="inputId" on the input. This makes forms accessible for screen readers and lets users click the label to focus the input.

🏛️ Semantic HTML

Semantic elements describe the meaning of their content — not just how they look. They make pages more accessible, SEO-friendly, and readable by humans and machines alike.

🤔
Semantic vs Non-Semantic A <div> or <span> says nothing about the content inside. A <nav> or <article> tells both the browser and developer exactly what it contains.
🏠

<header>

The introductory section of a page or section — typically contains logo, navigation, and hero content.

🧭

<nav>

A group of navigation links. Screen readers announce this as the navigation landmark.

📰

<main>

The primary content of the page. There should only be one <main> per page.

📄

<article>

Self-contained content that could be syndicated independently — a blog post, news article, or forum post.

📦

<section>

A thematic grouping of content, typically with a heading. Different from <div> — it has semantic meaning.

📎

<aside>

Content tangentially related to the main content — sidebars, pull quotes, or related links.

🦶

<footer>

Bottom section of a page or section — copyright, links, contact info.

🕐

<time>

Represents a date/time value. The datetime attribute gives it a machine-readable format.

Semantic Page Layout
<body>

  <header>
    <nav>
      <a href="/">Home</a>
      <a href="/about">About</a>
    </nav>
  </header>

  <main>
    <article>
      <h1>My Blog Post</h1>
      <time datetime="2025-01-15">January 15, 2025</time>
      <section>
        <h2>Introduction</h2>
        <p>Article content here...</p>
      </section>
    </article>

    <aside>
      <h3>Related Posts</h3>
    </aside>
  </main>

  <footer>
    <p>© 2025 My Website</p>
  </footer>

</body>

📊 HTML Tables

Tables are for displaying structured, tabular data — think spreadsheets, schedules, or comparison charts. Never use tables for page layout!

🗂️

<table>

The root container for all table content.

🧢

<thead> / <tbody> / <tfoot>

Semantic groupings for the header, body, and footer rows of a table.

➡️

<tr>

Table Row — wraps all cells in a single horizontal row.

🏷️

<th>

Table Header cell — bold and centered by default. Use for column/row labels.

📦

<td>

Table Data cell — holds the actual content of the table.

↔️

colspan / rowspan

Allow a cell to span multiple columns or rows — like merging cells in Excel.

Full Table Structure
<table>

  <!-- Table head -->
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>

  <!-- Table body -->
  <tbody>
    <tr>
      <td>Yehia</td>
      <td>22</td>
      <td>Cairo</td>
    </tr>
    <tr>
      <td>Sara</td>
      <td colspan="2">Unknown</td>
    </tr>
  </tbody>

  <!-- Table footer -->
  <tfoot>
    <tr>
      <td colspan="3">Total: 2 rows</td>
    </tr>
  </tfoot>

</table>
⚠️
Tables are NOT for Layout In the early web, developers used tables to create page layouts. This is a bad practice — use CSS Flexbox or Grid for layout, and <table> only for actual data.

🎬 Multimedia Elements

HTML5 brought native support for video, audio, and embedded content — no plugins required. These elements make the web rich and engaging.

🎥

<video>

Embeds a video player. Supports controls, autoplay, loop, and muted attributes.

🎵

<audio>

Embeds an audio player. Same attributes as <video>. Supports MP3, OGG, WAV formats.

🔊

<source>

Provides multiple file formats inside <video> or <audio>. Browser picks the first supported one.

🖼️

<iframe>

Embeds another HTML page or external content (YouTube, Google Maps, etc.) inside your page.

🎨

<canvas>

A drawable surface controlled by JavaScript. Used for animations, games, and data visualizations.

🗺️

<picture>

Responsive images — serve different image sizes or formats depending on screen size or browser support.

Video, Audio & iFrame
<!-- Video with controls -->
<video controls width="640" poster="thumbnail.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Your browser does not support video.
</video>

<!-- Audio player -->
<audio controls>
  <source src="song.mp3" type="audio/mpeg">
</audio>

<!-- Embed YouTube video -->
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560"
  height="315"
  allowfullscreen
></iframe>

<!-- Responsive image with fallback -->
<picture>
  <source srcset="img.webp" type="image/webp">
  <img src="img.jpg" alt="Description">
</picture>

♿ Accessibility (a11y)

Accessibility means making your website usable by everyone — including people who use screen readers, keyboard navigation, or have visual impairments.

🌍
1 Billion People Over 1 billion people worldwide live with some form of disability. Accessible websites are good practice and legally required in many countries.
🏷️

aria-label

Provides an accessible name for elements that lack visible text — like icon buttons.

👁️

aria-hidden

Hides decorative elements from screen readers so they don't clutter the experience.

🎭

role

Assigns a semantic role — role="button", role="alert", role="dialog".

⌨️

tabindex

Controls keyboard tab order. 0 = natural order, -1 = JS only.

🔤

Meaningful alt

Always provide meaningful alt text. Empty alt="" tells screen readers to skip decorative images.

🔗

lang

<html lang="en"> tells screen readers which language to use for pronunciation.

Accessibility Examples
<!-- ✅ Icon button with label -->
<button aria-label="Close menu">
  <span aria-hidden="true">✕</span>
</button>

<!-- ✅ Decorative image -->
<img src="divider.svg" alt="" aria-hidden="true">

<!-- ✅ Input with description -->
<label for="email">Email</label>
<input type="email" id="email"
       aria-describedby="hint">
<span id="hint">We never share your email.</span>

<!-- ❌ Bad: No alt, div as button -->
<img src="btn.png">
<div onclick="submit()">Click me</div>

🖼️ Responsive Images

Modern websites serve different images to different devices — small for phones, large for desktops. This saves bandwidth and improves performance significantly.

📱

srcset

Multiple sources with width descriptors. Browser picks the best one automatically.

📐

sizes

Tells the browser what size the image will display at different layout widths.

🖼️

<picture>

Serve completely different images or formats (WebP/AVIF) based on media queries.

⏳

loading="lazy"

Delays off-screen images from loading until the user scrolls near them.

🔄

decoding="async"

Decodes the image off the main thread so it doesn't block rendering.

📦

WebP / AVIF

Modern formats that are 25–50% smaller than JPEG/PNG with the same quality.

Responsive Image Techniques
<!-- srcset: serve different sizes -->
<img
  src="hero-800.jpg"
  srcset="hero-400.jpg 400w,
          hero-800.jpg 800w,
          hero-1200.jpg 1200w"
  sizes="(max-width:600px) 100vw, 800px"
  alt="Hero image"
  loading="lazy"
  decoding="async"
>
